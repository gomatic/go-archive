# go-archive

Stream-oriented `tar.gz` archiving: `Create` (paths → `io.Writer`), `Extract` (`io.Reader` → dir, with zip-slip protection), and `List`. Extracted from `gomatic/ssh-tgzx`'s `internal/archive`.

- Package `archive`, generic and dependency-free (stdlib only; testify for tests). Every failure wraps an `errs.Const` sentinel from [gomatic/go-error](https://github.com/gomatic/go-error) (`ErrCreateArchive`, `ErrExtract`, …) via `.With(cause, args…)`, matchable with `errors.Is`. The tar/gzip writer chain is owned by an unexported `creator` and the reader by an `extractor`, so the stateful stdlib pointers live in struct fields rather than threaded as parameters (keeping `yze/ptrparam` clean); `tar.Header` is passed by value and entry bodies as `io.Reader`. Path/name params use the `filePath`/`linkName` named domain types.
- Gate: gofumpt, vet, staticcheck, govulncheck, stickler (yze suite), gocognit ≤ 7, 100% coverage (incl. `FuzzExtract`/`FuzzList` over arbitrary `tar.gz` bytes). Shared config (`Makefile`, `.golangci.yaml`, `.github/`, …) is owned by `nicerobot/tools.repository` — never edit in-tree; use `Makefile.local` or `.stickler.yaml`.
- **`.stickler.yaml` divergence (durable):** this repo excludes one gosec rule — `G305` (zip-slip) — because `extractEntry` guards every entry _and_ every symlink target with `withinDir` before any write (tested by `TestExtract_PathTraversal` and `TestExtract_EscapingSymlinkIsRejected`) and gosec flags the unavoidable `filepath.Join(dest, header.Name)` regardless. The exclude lives in `.stickler.yaml` (`config.golangci-lint.linters.settings.gosec.excludes.add: [G305]`), which stickler deep-merges onto the distributed `.golangci.yaml`. Unlike an in-tree `.golangci.yaml` edit, this **survives `tools.repository` distribute pushes** — no manual re-add needed. (Even-more-durable fix would be excluding `G305` upstream in `tools.build`, like `G115`.)
